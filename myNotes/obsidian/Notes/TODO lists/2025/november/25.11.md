# Idea for final project
Как по мне, хорошая идея для этого финального проекта — система заметок. Почему:

- **Скорость разработки.** Текст — одна из самых легковесных единиц данных для хранения. Ведь текст — это набор символов, а символ — байт (или несколько байтов) с договорённостью о том, как его выводить. Нам не придётся выдумывать сложные алгоритмы для эффективного хранения данных. Мы можем использовать, например, алгоритм deflate и уже хранить данные максимально эффективно. Поскольку у нас на счету всего 22 дня, простота реализации критически важна.
    
- **Простота бизнес-логики.** Для системы заметок первой версии нужно реализовать довольно мало фич:
    
    1. **Поддержка Markdown.** Рендеринг Markdown через превращение в HTML.
        
    2. **Поддержка автосохранения.** Например, можно реализовать простой буфер, который раз в некоторое время шлёт запрос на обновление.
        
    3. **Возможность переключаться между заметками через GUI.** Тут вообще всё просто.  
        **Никаких папок, плагинов. Мы делаем первую версию, и подобного функционала с головой хватает!! Не забываем про KISS**
        
- **Практическая ценность.** Система заметок — довольно важный и востребованный инструмент для современного мира. Многие понимают, насколько важно организовывать мысли для продуктивности и саморефлексии. Создав систему заметок, мы можем создать фундамент для будущего клона Notion или Obsidian со своими фичами.
