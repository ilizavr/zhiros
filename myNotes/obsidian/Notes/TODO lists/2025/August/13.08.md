## что может работать

есть интерфейс Render который выводит на страничку то что передали в метод Render, таке интерфейс Render требует типа контента. вот как выглядит интерфейс:
```go
type Render interface {
	// Render writes data with custom ContentType.
	Render(http.ResponseWriter) error
	// WriteContentType writes custom ContentType.
	WriteContentType(w http.ResponseWriter)
}
```

- в gin есть тип data который может представлять что угодно и он может рендериться. можно попробовать Data для md. вот как выглядит структура:
  ```go
type Data struct {
	ContentType string
	Data        []byte
}
```
- еще есть тип MsgPack который тое может сайти за рендер markdown. вот как выглядит структура:
```go
type MsgPack struct{
Data any
}
```

# как делать ruslandev
для этого используй пакет gorm который позволяет обращаться к sqlite. также для sqlite нужен специальный драйвер который мы тоже установим,  вот подробности:

### как скачать драйвер sqlite
```sh
go get github.com/glebarez/sqlite
```

### какие методы в gorm используются

- Open - создает сокет между приложением и бд. короче открывает соединение. требует драйвер и конфиг. пример см [тут](https://pkg.go.dev/github.com/glebarez/sqlite#section-readme)
- Create - добавляет в **СУБД** то, что ты передал в параметр. возращает тип DB, у которого есть поле Error. см пример [тут](https://github.com/horhhe/DCAE/blob/main/internal/handlers/auth.go). метод не нужен
- Exec - выполняет любой sql запрос, любооой!
