# алгоритм реализации пайплайна

- в отдельной функции, проверяем, является ли команда пайплайном(через цикл)
- if является, создаем fork для первой команды 
- внутри этого fork создаем пайп для последущего общения через функцию pipe()
- потом создаем dup2 так, чтобы в процессе нашей первой команды весь текст из stdout переносился в fd[1] (это конец пайпа для записи) 
- после, создаем fork для 2 команды
- если текущий процесс - 2 команда, читаем из fd[0] (конец папйпа для чтения)
-  через write записываем буфер в stdin второй команды
- если текущий процесс - первая команда, то выполняем первую команду через execvp. если она имеет какой то вывод, он перенесется в конец пайпа для записи(это  действие было еще в начале алгоритма)
