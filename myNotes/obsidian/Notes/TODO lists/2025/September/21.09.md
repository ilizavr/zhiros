# конспект по elastic search(потом создай отдельный конспект)

## ключевые понятия
cluster - это единица, которая содержит 1 или более нодов. ноды общаются между собой чтобы координировать операции,  передеавать и делиться информацией и предотврощать ошибки

#### Node

- A node is a single instance of Elasticsearch running on a machine within a cluster.  
    
- Each node stores a part of the data and participates in the cluster's indexing and search capabilities.  
    
- Nodes can be categorized into different roles, such as master-eligible nodes, data nodes, and coordinating nodes.

### 2. Indexing and Data Model

Elasticsearch organizes and stores data in the form of documents within indices. Documents are [JSON](https://www.geeksforgeeks.org/javascript/javascript-json/) objects that contain data and metadata associated with the data.

#### Index

- An index is a grouping of documents that share common characteristics.  
    
- Indices are similar to [databases](https://www.geeksforgeeks.org/dbms/what-is-database/) in traditional [SQL](https://www.geeksforgeeks.org/sql/sql-tutorial/) databases.  
    
- Each document within an index has a unique identifier (_id) and is stored in a structured format using JSON.

#### Document

- A document is a basic unit of information in Elasticsearch.  
    
- Documents are represented as [JSON](https://www.geeksforgeeks.org/javascript/javascript-json/) objects and contain data fields and their corresponding values.  
    
- Elasticsearch automatically indexes each field within a document and allowing for efficient searching and retrieval.
