# my think about adding pop instruction

 When i've seen how Javac generates JBT based on java source code, i decided: the pop instruction in my JIT is useless because i'm have store/load instructions that do same work. As example,see simple source java code:
```java
class Square {

static int square(int num) {

int x = 5;

  

int foo = x * 5;

return num * num;

}

}
```

and after this,its correspond byte-code:
```go
class Square {

Square();

0: aload_0

1: invokespecial #1 // Method java/lang/Object."<init>":()V

4: return

  
  

static int square(int);

0: iconst_5

1: istore_1

2: iload_1 // this a key instruction

3: iconst_5

4: imul

5: istore_2

6: iload_0

7: iload_0

8: imul

9: ireturn

  

}
```

As you may see,load instruction loads value from variable Numner 1(from pool) to the stack. Therefore i'm decided delete pop instruction from list of JIT instructions
