# запуск gdb
Запустить gdb довольно легко, для запуска не требуются какие-то особые параметры. Для запуска отладчика используй синтаксис. Программа должна быть скомпилирована с флагом `-g`, чтобы активировать полномасштабно функции дебага:
```bash
gdb /path/program.c
```

дело в том что по умолчанию компилятор уже генерирует базовые функции дебага. Но этих функций вообще недостаточно, так как по умолчанию эти функции описывают базовую работу прогграмы. флаг `-g` говорит компилятору добавить функции дебага(labels) для каждой строки. Чтобы дебаггер мог смотреть в таблицу символов и тд

# команды gdb

## общие
 В этом разделе ты узнаешь о самых базовых командах которые в целом покрывают большинство базовых случаев:
 
- `break line` или `b line` - создать точку останова на строке line. Также заместо line может быть и адрес и функция
    
- `info info_for_out` или `i info_for_out` - вывести информацию о info_for_out. Для info_for_out доступны следующие значения:
    
1. `break` - вывести все созданные точки останова
        
2. `registers` - вывести все текущие используемые регистры функцией
        
3. `frame <NUM>` - вывести фрейм с номером NUM. Если NUM не определен, выводится текущий фрейм функции. Подробнее смотри [[конспект по gdb#что выводит команда i frame в gdb|тут]] 
        
4. `locals <VAR>` - вывести значение локальной переменной VAR в текущем стеке функции. Если VAR  не указан,выводятся все значения локальных переменных
        
5. `args` - вывести значения всех аргументов в текущей функции
        
- `run <args>` или `r <args>` - запустить программу до текущей точки останова с аргументами (или без)
    
- `continue` или `c` - продолжить выполнение программы после точки останова
    
- `step` или `s` - выполнить следующую строчку программы с заходом в функцию
    
- `next` или `n` - выполнить следующую строчку без захода в функцию
    
- `print` или `p` - вывести значение переменной
    
- `backtrace` или `bt` - вывести стек вызовов
    
- `x` - просмотр содержимого памяти по адресу. Она имеет некоторые возможности в форматировании:
    
    1. `o` - вывести значение в восьмеричной системе
        
    2. `x` - вывести значение в hex
        
    3. `t` - вывести значение в binary
        
    4. `d` - вывести значение в decimal
        
    5. `i` - вывести значение как инструкция процессора
        
    6. `c` - вывести значение как char
        
		7. `s` - вывести значение как string
    также команда x имеет некоторые возможности в размере выводимого блока:
1. `b` - байт
2. `h` -  полуслово(2 байта)
3. `w` - слово(4 байта)
4. `g` - квад(8 байт)
также команда x позволяет определять количество блоков для вывода. В общем вызов команды x выглядит как:
`x/CSF <VAR>` - где C - это количество блоков,S - размер выводимого блока,F - формат вывода,VAR - переменная

- `ptype` - просмотр типа переменной
    
- `h` или `help` - просмотр справки по команде
    
- `q` или `quit` - выход из программы
- `handle SIGNAL <DO...>` - как либо обрабатывать сигнал SIGNAL. обозначение DO... означает какие действия предпринимаются при получении сигнала. сигнал может быть al, что значит делать что то со всеми сигналами. Вот неокторые действия:
1.  stop/nostop - остановить или не остановить прогграму при получении сигнала
2. print/noprint - выводить или не выводить сообщение о получении сигнала
3. pass/nopass - передавать или не передавать сигнал обработчику сигналов прогграмы
**Также имеются некоторые подробности:**  
`&` - знак взятия адреса. Используется в print и x



## Диззасемблирование
 В этом разделе вы узнаете про команды дизасемблирования которые  очень сильно помогут тебе при дизасемблировании софта:
 - `disassembly <FUNC>` - дизассемблирует код функции FUNC в виде ассемблера AT&T синтаксиса. если FUNC не указан, выводится ассемблер текущей функции. также есть опции \m,\r. Вот что они делают:
 1. опция \m выводит диссасемблер с соответствующим кодом на исходном языке
 2. опция \r выводит диссасемблер с соответствующим машинным кодом
   Также помимо простого дизасемблирования имени функции имеются и другие возможности:

   1.вывод адреса. Вместо имени функции можно подставить конкретный адрес НО только внутри функции
   2.вывод инструкций между [START] и [END]. Можно выводить инструкции только между стартовым и конечным адресом
   3.вывод инструкций начиная с FUNC + LEN. Можно выводить инструкции начиная с начала функции + какое то смещение. Грубо говоря начать дизассемблирование на LEN строчке позже
   - `x/Ni addr` - да я описывал команду x, но это было в общем а сейчас опишу подробности нужные для дизассемблера. Команда x нужна тогда, когда нужно посмотреть произвольный адрес с инструкциями. N - это количество инструкций которое нужно вывести. addr - адрес откуда начинать дизассемблирование
   - `info functions` - да это опять общая команда но остановимся на подкоманде funcitons. Эта подкоманда выводит все функции которые присутствуют в бинарнике включая  start



## многопоточность
 В этом разделе подробно поговорим про очень полезные для многопоточности команды:
 - `info threads` - показывает все потоки текущей прогграмы. Если подробно,то эта команда показывает:
 1. все потоки прогграмы
 2. id потока для каждого
 3. текущую функцию которая выполняется в каждом потоке. Звездочкой показывается активный поток с которым ты работаешь в gdb
 - `thread <ID>` - переключает контекст выполнения gdb на поток с указанным ID



# что выводит команда i frame в gdb

Когда ты вводишь "i frame", ты видишь информацию о стековом фрейме. Почти все данные понятны, но остановимся на самом важном. Смотри:

```html
(gdb) i frame
Stack level 0, frame at 0x7fffffffd5c0:
 rip = 0x600cb6 in test_work_for_ret (<exploit.c:9); saved rip = 0x4646
 called by frame at 0x7fffffffd5c8
 source language c.
 Arglist at 0x7fffffffd5b0, args: 
 Locals at 0x7fffffffd5b0, Previous frame's sp is 0x7fffffffd5c0
 Saved registers:
  rbp at 0x7fffffffd5b0, rip at 0x7fffffffd5b8
```
Теперь разберем смысл всех rip'ов:

- **Первый rip** (`rip = 0x600cb6`) - указывает на следующую инструкцию, которая будет выполнена
    
- **Второй rip** (`saved rip = 0x4646`) - значение указателя возврата (return instruction pointer)
    
- **Третий rip** (`rip at 0x7fffffffd5b8`) - адрес в стеке, где хранится сохраненный rip
