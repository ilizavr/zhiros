# переменные, использующиеся функцией
- opterr - переменная которая нужна для обработки ошибок. если значение этой переменной не 0, getopt выводит сообщение об ошибке в stderr если встречается с неизвестной опцией или  обязательным пропущенным аргументом. это поведение по умолчанию. если значение переменной 0 - getopt не скажет об ошибке, но вернет знак ?, который говорит о том что есть ошибка
- optopt - переменная которая нужна для того чтобы  посмотреть какую неправильную опцию ввел пользователь. когда getopt встречает неизвестную опцию, или опцию с пропущеным обязательным аргументом, getopt записывает эту опцию в optopt
- optind - переменная которая нужна для получения индекса следущей опции из массива argv, которую getopt будет обрабатывать. после того как getopt оработал все аргументы опции, можно использовать optind  для определения оставшихся аргументов не опций. изначальное значение optind - 1 
- optarg - переменная которая нужна для получения значения аргумента опции, для тех опций которые принимают аргументы

# функция getopt
## декларация
```c
int getopt(int argc, char *const *argv, const char *options)
```

## описание


функция getopt возращает имя опции в виде типа char, из массива argv и счетчика argc. обычно getopt вызывают из main

третий Аргумент options — это строка, содержащая  опции, которая может принимать прогграма. Символ  опции  может сопровождаться двоеточием (‘:’), что указывает на необходимость обязательного аргумента для данной опции. Если за символом параметра следуют два двоеточия (‘::’), аргумент для этой опции  необязателен; это расширение GNU.

getopt имеет три способа обработки аргументов, следующих за элементами argv, не являющимися опциями. Специальный аргумент '--' во всех случаях принудительно завершает сканирование опций.


- По умолчанию содержимое argv переставляется при его сканировании таким образом, чтобы все не-опции оказывались в конце. Это позволяет указывать параметры в любом порядке, даже в программах, которые не были написаны для этого.
- Если строка аргумента параметров начинается с дефиса ('-')(если 1 элемент options -), это обрабатывается особым образом. Это позволяет возвращать аргументы, не являющиеся опциями, как если бы они были связаны с символом параметра '\1'.
- POSIX требует следующего поведения: первая не-опция останавливает обработку опций. Этот режим выбирается либо установкой переменной окружения `POSIXLY_CORRECT`, либо добавлением знака плюс ('+') в начало строки аргумента параметров.

Функция getopt возвращает символ опции для следующего параметра командной строки. Если аргументы опции больше недоступны, функция возвращает `-1`. Могут быть и другие аргументы, не являющиеся опциями; для проверки этого необходимо сравнить внешнюю переменную `optind` с параметром argc.

Если у опции есть аргумент, `getopt` возвращает аргумент, сохраняя его в переменной optarg. Обычно не требуется копировать строку `optarg`, поскольку она является указателем на исходный массив argv, а не на статическую область, которая может быть перезаписана.

Если `getopt` находит символ опции в argv, который не был включён в options, или отсутствующий аргумент опции, функция возвращает `?` и присваивает внешней переменной `optopt` значение фактического символа опции. Если первый символ в options — двоеточие (‘:’), то `getopt` возвращает `:’ вместо `?`, указывая на отсутствующий аргумент опции. Кроме того, если внешняя переменная `opterr` не равна нулю (что является значением по умолчанию), `getopt` выводит сообщение об ошибке.
